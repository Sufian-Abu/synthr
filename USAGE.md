# Using Synthr

How engineers consume the gateway — REST, the SDKs, and what each docs URL is for.

---

## 1. Endpoints & docs surfaces

With the gateway running (`http://localhost:8000` by default):

| URL | What it is | Use it to |
|---|---|---|
| `/dashboard` | HTMX usage dashboard | watch requests, cost, cache hits, guardrail blocks/redactions |
| `/redoc` | **ReDoc** API reference (read-only) | read/share the API; the single human-facing doc |
| `/health` | liveness + configured features/providers | health checks |
| `/` | index of the above | quick links |

The API reference is auto-generated by FastAPI from the route signatures + Pydantic models —
you don't maintain it. (Swagger UI is disabled; ReDoc renders from the OpenAPI schema served
internally at `/openapi.json`.) To test endpoints, use `curl`, the SDKs, or the examples below.

---

## 2. Auth — project keys

Send your key in the `X-Project-Key` header.

- **Secret key** `sk_proj_…` — backend / REST / scripts. Full access.
- **Public key** `pk_proj_…` — browsers. Only works from an allowed origin and only for
  features marked `frontend_safe: true`.
- Optional `X-User-Id` header — used for per-user rate limits.

Generate a key: `synthr keygen` (or `synthr keygen --public`). Add it under a project in
`synthr.config.yaml`.

## 3. REST (any language)

```bash
curl -X POST http://localhost:8000/v1/fillForm \
  -H "Content-Type: application/json" \
  -H "X-Project-Key: sk_proj_demo_secret" \
  -d '{"fields":[{"name":"brand","type":"string"},{"name":"size","type":"number"}],
       "context":"Nike Air Max size 10"}'
```

Response envelope (every feature):
```json
{ "data": { ... }, "meta": { "feature": "...", "provider": "...", "cached": false,
                             "request_id": "req_…", "usage": { "prompt_tokens": 0, "completion_tokens": 0 } } }
```
Errors: HTTP 4xx/5xx with `{ "error": { "code": "...", "message": "...", "retry_after_seconds": 0 } }`.
Codes: `invalid_key · origin_not_allowed · feature_not_allowed · rate_limited · guardrail_blocked · provider_error · invalid_input · internal_error`.

### Endpoints
| Endpoint | Body |
|---|---|
| `POST /v1/fillForm` | `{ "fields":[{"name","type","options?"}], "context": "…", "locale?": "en" }` |
| `POST /v1/summarize` | `{ "text": "…", "max_words?": 20 }` |
| `POST /v1/translate` | `{ "text": "…", "target_lang": "Spanish" }` |
| `POST /v1/image` | `{ "prompt": "…", "size?": "1024x1024", "n?": 1 }` (backend-only) |
| `POST /v1/removeBackground` | `{ "image": "<base64>" }` or `{ "image_url": "https://…" }` |

## 4. Python SDK

Location: [`sdk/python`](sdk/python/). Not yet on PyPI — install from source for now:

```bash
pip install ./sdk/python          # later: pip install synthr-sdk
```

```python
from synthr import AI

ai = AI(key="sk_proj_...")         # url defaults to http://localhost:8000 or $SYNTHR_URL
ai.fill_form(fields=[{"name":"brand","type":"string"}], context="Nike Air Max")
ai.summarize(text="…", max_words=20)
ai.translate(text="Good morning", target_lang="Spanish")
ai.image(prompt="a red shoe")
ai.remove_background(image_url="https://…")
ai.run("custom_feature", {"foo": "bar"})   # escape hatch for any feature
```

Async is identical with `AsyncAI` + `await`. Errors raise `SynthrError`
(`.code`, `.message`, `.status`, `.retry_after`).

## 5. TypeScript / JavaScript SDK

Location: [`sdk/typescript`](sdk/typescript/). Build from source for now:

```bash
cd sdk/typescript && npm install && npm run build   # later: npm install synthr-sdk
```

```ts
import { AI } from "synthr-sdk";   // or the built ./dist for now

// Browser: PUBLIC key (pk_proj_…). Node/backend: secret key (sk_proj_…).
const ai = new AI({ url: "http://localhost:8000", key: "pk_proj_demo_public" });

const { values } = await ai.fillForm(
  [{ name: "brand", type: "string" }, { name: "size", type: "number" }],
  "Nike Air Max size 10",
);
await ai.translate("Good morning", "Japanese");
```

Works in the browser and Node 18+. Errors throw `SynthrError`.
